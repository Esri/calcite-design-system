calcite-nav-menu

The calcite-nav-menu component is placed in the *-menu-* slots.

logo user primary-menu-start primary-actions-start primary-menu-center primary-actions-end primary-menu-end secondary-actions-start secondary-actions-end secondary-menu-start secondary-menu-end tertiary-actions-start tertiary-actions-end tertiary-menu-start tertiary-menu-end

The calcite-nav-menu component can be used within defined slots in calcite-nav, or on its own within other portions of the UI.

Properties

layout: horizontal / vertical - this is the display mode of the nav-menu. most cases will leave unset (horizontal). A single nav-menu can be used in vertical in a sheet, or shell, for example

The calcite-nav-menu component appends items into an overflow dropdown when the width of the nav-menu is too narrow. You can customize, or disable this functionality with properties. Order is not changed - if an active tab is appended to the overflow menu, the active prop will be added to the "ellipsis dropdown"

overflow-type: (future feature) "start" (default)"end""follw"(disable overflow functionality - should we support this?)

min-items: Ensure this many items are persisted (as specified by overflow-type value)

max-items: Limit items shown to this amount and append rest to overflow menu (as specified by overflow-type. Can be used with breadcrumb
